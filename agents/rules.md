🤖 AI Coding Agent Rules (Telegram Bot Development)

1. Code Quality & Standards
   Always follow industry-standard best practices and clean coding principles.
   Ensure code is readable, maintainable, and well-documented.
   Use consistent naming conventions and proper formatting.
2. Architecture & Design
   Apply Object-Oriented Programming (OOP) principles where appropriate.
   Follow SOLID principles to ensure scalability and maintainability.
   Design the system with modularity and clear separation of concerns.
3. File & Project Structure
   If any file exceeds 500 lines of code, refactor and split it into smaller modules.
   Each file/module must have a single clear responsibility.
   Organize code into logical layers (e.g., handlers, services, models, utilities).
4. Reusability & Abstraction
   Write reusable components and avoid code duplication (DRY principle).
   Identify repeated logic and extract it into reusable functions, classes, or modules.
   Use proper abstraction and encapsulation to improve flexibility.
   Ensure components are loosely coupled and highly cohesive.
5. Working with Existing Code
   Always analyze and follow the existing codebase structure and conventions.
   Do not introduce inconsistent patterns or unnecessary rewrites.
   If reusable logic is found, refactor and abstract it into shared components.
   Extend existing systems instead of duplicating functionality.
6. Telegram Bot Best Practices
   Structure bot logic into handlers, commands, and services.
   Clearly separate API interaction, business logic, and bot commands.
   Implement proper error handling and logging.
7. Performance & Reliability
   Optimize performance without sacrificing readability.
   Handle edge cases and unexpected inputs gracefully.
   Ensure the bot is stable and scalable.
8. Cost-Efficiency (High Priority)
   Always prioritize free, open-source, and no-cost solutions.
   Avoid paid APIs, services, or tools unless absolutely necessary.
   Prefer self-hosted or free-tier options whenever possible.
   Clearly highlight any potential costs and suggest free alternatives.
9. Documentation & Clarity
   Add clear comments for complex logic.
   Provide concise explanations for key components.
   Include setup and deployment instructions when generating full projects.
